# Evaluate Progress - הערכת התקדמות

בדוק את עבודת התלמיד ותן משוב מפורט.

## תהליך הערכה

### 1. קבלת הגשה
```
/evaluate-progress [path-to-exercise]
```

### 2. בדיקות אוטומטיות

#### בדיקת קיום קבצים:
```bash
# האם כל הקבצים הנדרשים קיימים?
for file in required_files; do
  test -f "$file" || echo "חסר: $file"
done
```

#### בדיקת תוכן:
```bash
# האם התוכן תואם לדרישות?
grep -q "expected_pattern" file.txt
```

#### בדיקת תקינות:
```bash
# האם הקוד רץ ללא שגיאות?
python script.py 2>&1 | grep -i error
```

### 3. קריטריונים להערכה

| קריטריון | משקל | תיאור |
|----------|------|--------|
| שלמות | 40% | כל הדרישות מולאו |
| נכונות | 30% | הפתרון עובד |
| סגנון | 20% | קוד קריא ומסודר |
| יצירתיות | 10% | פתרון מקורי/יעיל |

### 4. פורמט משוב

```markdown
## 📊 הערכת תרגיל: [שם]

### סטטוס: ✅ הושלם / ⚠️ דורש תיקון / ❌ לא הושלם

### ציון: X/100

### מה עבד טוב:
- ✅ נקודה 1
- ✅ נקודה 2

### מה צריך לשפר:
- ⚠️ הערה 1
- ⚠️ הערה 2

### הצעות לשיפור:
1. הצעה ספציפית
2. הצעה ספציפית

### הצעד הבא:
[מה ללמוד/לתרגל עכשיו]
```

## מעקב התקדמות

### מבנה קובץ מעקב
```markdown
# התקדמות: [שם התלמיד]

## Tier 1: מבוא
- [x] שיעור 1: מה זה Claude Code
- [x] שיעור 2: פקודות בסיסיות
- [ ] שיעור 3: פרויקט ראשון

## תרגילים שהושלמו:
| תרגיל | תאריך | ציון |
|-------|--------|------|
| ארגון קבצים | 2024-01-15 | 85 |

## הערות:
- חזק בהבנה תיאורטית
- צריך יותר תרגול מעשי
```

## סוגי משוב

### משוב חיובי (הצלחה):
```
מעולה! הצלחת להשלים את התרגיל.

נקודות חזקות:
- הקוד מסודר וקריא
- פתרון יצירתי

אתה מוכן לשיעור הבא!
```

### משוב בונה (צריך שיפור):
```
עבודה טובה, יש כמה דברים לשפר:

1. [בעיה ספציפית]
   → הצעה לתיקון

2. [בעיה ספציפית]
   → הצעה לתיקון

נסה שוב ותראה לי את התוצאה.
```

### משוב לאי-הצלחה:
```
נראה שנתקעת. זה בסדר!

בוא נחזור צעד אחורה:
1. קרא שוב את [חלק X] בשיעור
2. נסה את [תרגיל קל יותר] קודם
3. אם עדיין קשה - שאל שאלות

אני כאן לעזור 😊
```

## שימוש

```
/evaluate-progress exercises/tier-1/exercise-1/
/evaluate-progress --quick exercises/test.py
```

$ARGUMENTS: [path] [--quick for brief check]
